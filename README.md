# 편의점 데이터베이스 관리 시스템 (Convenience Store DB System)

본 프로젝트는 편의점 유통 환경에서 발생하는 판매, 재고, 발주, 상품/공급업체, 고객 정보를 유기적으로 관리하고 통계를 분석할 수 있는 SQLite 기반의 CLI(Command Line Interface) 시스템입니다.

 ### 2. 데이터베이스 파일 제출에 관한 중요 안내
- 과제 요구사항에 의거하여 실제 데이터베이스 파일인 `convenience.db`는 제출 압축 파일(.zip)에 포함하지 않았습니다.
- 전체 데이터가 적재된 정식 데이터베이스 파일은 아래 Google Drive 링크를 통해 다운로드하실 수 있습니다.

* **Google Drive 다운로드 링크:** [여기에 convenience.db 공유 링크 입력]

## 3. 데이터베이스 구축 안내

본 프로젝트는 채점관 환경에 따라 데이터베이스를 연동할 수 있도록 **두 가지 방법**을 모두 지원합니다. 편하신 방법으로 하나만 선택하여 데이터베이스를 준비해 주십시오.

### 방법 A) 구글 드라이브에서 정식 DB 다운로드 (추천)
1. 위의 구글 드라이브 링크에서 완성된 `convenience.db` 파일을 다운로드합니다.
2. 다운로드한 파일을 `src/` 폴더 안으로 이동시켜 `main.py`와 **완전히 동일한 위치**에 배치합니다.
3. 곧바로 **[STEP 2: 메인 CLI 시스템 실행]** 명령어를 입력하여 실행합니다.

### 방법 B) 파이썬 스크립트로 로컬에서 새 DB 자동 생성 (make_db.py)
1. 네트워크 제한 등으로 구글 드라이브 다운로드가 어려울 경우 활용합니다.
2. `src/` 폴더 내의 복제 코드를 실행하여 로컬 환경에서 즉석으로 DB를 빌드합니다. [cite: 72]
3. 실행 즉시 프로그램 내부 코드가 작동하여 필수 테이블 스키마를 빌드하고 테스트용 초기 샘플 데이터를 로컬에 자동으로 구축해 줍니다. [cite: 48]

 ### 4. 인터페이스 실행 방법 (How to Run)

 본 프로그램은 파이썬 내장 표준 라이브러리(`sqlite3`, `os`, `datetime`)만을 사용하여 구현되었으므로, 별도의 외부 패키지 설치(`pip install`) 없이 즉시 실행이 가능합니다. 압축을 해제한 후, 터미널(Terminal)에서 아래의 순서대로 명령어를 입력하여 실행해 주십시오.

### STEP 1: 평가용 로컬 데이터베이스 환경 구축 (택일)
#### [선택 1] 구글 드라이브에서 다운로드하여 실행하는 경우 (방법 A)
- 다운로드한 `convenience.db` 파일을 `src/` 폴더 안에 위치시켰다면, **별도의 명령어 입력 없이** 즉시 아래의 **STEP 2**로 넘어갑니다.

#### [선택 2] 로컬에서 스크립트로 자동 생성하여 실행하는 경우 (방법 B)
- 채점관 환경에서 즉시 정상적인 쿼리 조회 및 트랜잭션 테스트가 가능하도록, 파이썬 스크립트를 수행하여 필수 테이블 스키마를 생성하고 초기 샘플 데이터를 로컬에 적재합니다. [cite: 48, 50]
bash
python src/make_db.py

스크립트 실행이 완료되면 src/ 폴더 내부에 convenience.db 파일이 자동으로 안전하게 생성됩니다.

### STEP 2: 메인 CLI 시스템 실행(필수)
데이터베이스 파일 준비(선택 1 또는 선택 2)가 완료되면 아래 명령어를 통해 메인 제어 인터페이스를 구동합니다. 시스템 내부적으로 sqlite_master 검증을 거쳐 필수 테이블이 정상 구비되었는지 자동으로 확인합니다.
bash
python src/main.py

### 5. 기능 평가를 위한 테스트 가이드 (Sample Data Guide)

본 시스템은 구현된 데이터베이스의 기능을 즉각적으로 평가할 수 있도록 make_db.py를 통해 연동된 전용 데이터셋을 제공합니다.
테스트 시 아래의 샘플 데이터를 입력하시면 즉시 결과를 확인하실 수 있습니다.

매장 ID 테스트 번호: S01 (서울시 매장), S02 (부산시 매장)
상품 바코드 테스트 번호: P001 (칠성사이다), P002 (신라면 블랙)
고객 및 회원 테스트 번호: C001 (이영희 회원, 연락처: 010-1111-2222)

추천 테스트 시나리오 
메인 메뉴 [2] 재고 관리 ➡️ [2] 매장별 재고 조회 선택 ➡️ 매장 ID에 S01 입력 시, 해당 매장의 칠성사이다 및 신라면 블랙의 실시간 재고와 판매가가 완벽히 매핑되어 출력됩니다.
메인 메뉴 [6] 통계 조회 ➡️ [1] 매장별 총매출 조회 선택 시, SUM과 GROUP BY 연산을 통해 각 매장별 누적 매출액이 집계되어 고성능 OLAP성 통계 데이터가 즉시 산출됩니다.

### 6. 인터페이스 설계 특징 명세서의 제약 요건을 준수하여 총 7개의 주 메뉴 인터페이스를 설계하였습니다.

- 판매 관리: 누적 판매 내역 및 영수증별 상세 내역 매핑 조회와 더불어, 실시간으로 매장 재고를 차감하고 회원 포인트를 적립하는 [POS 판매 처리] 트랜잭션을 지원합니다.
- 재고 관리: 매장별 전체 재고 현황 파악 및 실시간 안전 재고 관리를 위해 임의의 수치를 입력하여 추적하는 [재고 부족 상품 조회] 기능을 수행합니다.
- 발주 관리: 공급업체별 발주 내역 상세 조회 및 공급가 정보를 연동하여 데이터베이스에 실시간으로 반영하는 [신규 발주 등록] 프로세스를 제공합니다.
- 상품/공급업체 관리: 시스템 내 전체 상품 명세 검색, 특정 브랜드 검색 시 패턴 매칭(LIKE) 조회 및 업체별 취급 품목 조회가 가능합니다.
- 고객 관리: Supertype-Subtype 구조(Customer-Member/NonMember)를 완벽하게 반영하여 고객의 유형을 식별하고 , 비회원의 [신규 회원 등록 및 전환] 비즈니스 로직을 처리합니다.
- 통계 조회: 매장별 총매출 집계, 제품별 누적 판매 수량/금액 정렬, 회원별 기여도 평가를 위한 누적 금액 산출 및 자산 가치 평가를 위한 [매장별 재고 자산 금액 조회] 등 다중 조인 집계 쿼리를 제공합니다.
- 종료: 데이터베이스 커넥션을 안정적으로 close() 하고 메모리를 릴리즈한 뒤 시스템을 전면 종료합니다.
