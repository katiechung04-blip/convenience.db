# 편의점 데이터베이스 관리 시스템

## Convenience Store DB System

본 프로젝트는 편의점 유통 환경에서 발생하는 판매, 재고, 발주, 상품/공급업체, 고객 정보를 관리하고 통계 분석을 수행할 수 있는 SQLite 기반 CLI(Command Line Interface) 시스템입니다.

---

## 1. 프로젝트 디렉토리 구조

```text
팀이름_DB최종.zip
├── README.md
├── ERD_설명문서.pdf
└── src/
    ├── main.py
    └── sample_queries.sql
```

각 파일의 역할은 다음과 같습니다.

```text
README.md
- 시스템 개요 및 실행 방법 안내

ERD_설명문서.pdf
- E-R Diagram, Relational Schema, 설계 설명, 데이터베이스 다운로드 링크 포함 문서

src/main.py
- SQLite 데이터베이스를 읽어 CLI 인터페이스를 실행하는 메인 프로그램

src/sample_queries.sql
- 명세서의 주요 사용 시나리오를 검증하기 위한 샘플 SQL 쿼리 모음
```

실제 데이터베이스 파일인 `convenience.db`는 과제 요구사항에 따라 제출 압축 파일에 포함하지 않았습니다.

## 3. 실행 방법

본 프로그램은 Python 내장 라이브러리인 `sqlite3`, `os`, `datetime`만 사용하므로 별도의 외부 패키지 설치가 필요하지 않습니다.

### STEP 1. 데이터베이스 파일 배치

Google Drive 링크에서 `convenience.db` 파일을 다운로드한 뒤, `src` 폴더 안에 넣습니다.

```text
src/
├── main.py
├── sample_queries.sql
└── convenience.db
```

### STEP 2. 프로그램 실행

터미널에서 `src` 폴더로 이동한 뒤 다음 명령어를 실행합니다.

```bash
cd src
python main.py
```

실행 후 다음과 같은 메인 메뉴가 출력됩니다.

```text
=== 편의점 DB 시스템 ===
1. 판매 관리
2. 재고 관리
3. 발주/공급 관리
4. 상품/공급업체 관리
5. 고객 관리
6. 통계/테스트 쿼리/DBA 조회
7. 종료
```

## 4. 테스트용 주요 입력 예시

실제 DB에 저장된 값을 기준으로 테스트해야 합니다. 예시는 다음과 같습니다.

```text
매장 ID 예시:
1

상품 바코드 예시:
880100000001

고객 ID 예시:
1

고객 전화번호 예시:
010-1111-1111
```

POS 판매 처리, 매장별 재고 조회, 발주 등록 등의 기능을 테스트할 때 위와 같은 실제 DB 값을 입력하면 됩니다.

## 5. 인터페이스 설계 특징

본 시스템은 명세서의 조건에 맞추어 총 7개의 메인 메뉴로 구성하였습니다.

```text
1. 판매 관리
2. 재고 관리
3. 발주/공급 관리
4. 상품/공급업체 관리
5. 고객 관리
6. 통계/테스트 쿼리/DBA 조회
7. 종료
```

### 판매 관리

판매 내역 조회, 판매 상세 조회, POS 판매 처리를 제공한다. POS 판매 처리 시 `Sale`, `SaleDetail`에 판매 기록이 생성되고, 판매된 수량만큼 `StoreInventory`의 재고가 감소한다. 회원 고객인 경우 포인트가 적립된다.

### 재고 관리

전체 재고 조회, 매장별 재고 조회, 재고 부족 상품 조회를 제공한다.

### 발주/공급 관리

발주 내역 조회, 발주 상세 조회, 신규 발주 등록, 자동 재주문 발주 생성, 공급업체 발주 처리, 공급 처리 기록 조회 기능을 제공한다. 공급 처리가 완료되면 공급 기록이 저장되고 매장 재고가 증가한다.

### 상품/공급업체 관리

전체 상품 조회, 바코드로 상품 조회, 브랜드별 상품 조회, 공급업체 조회, 공급업체별 공급 가능 상품 조회 기능을 제공한다.

### 고객 관리

전체 회원 조회, 전체 비회원 조회, 전화번호로 고객 조회, 신규 회원 등록, 신규 비회원 등록, 고객 구매 내역 조회 기능을 제공한다. `Customer`는 회원과 비회원의 공통 정보를 저장하고, `Member`와 `NonMember`는 각각 고객의 하위 타입으로 구현하였다.

### 통계/테스트 쿼리/DBA 조회

매장별 인기 상품, 시·도별 인기 상품, 판매 실적 우수 매장, 제품 간 판매량 비교, 함께 구매된 상품 분석 등의 OLAP성 쿼리를 제공한다. 또한 관리자가 직접 `SELECT` 문을 입력하여 DB 내용을 확인할 수 있는 DBA 조회 기능을 제공한다.

---

## 6. 공급 처리 기록 테이블 안내

공급업체가 발주를 처리한 시각, 공급된 수량, 처리된 발주 내역을 기록하기 위해 `SupplyRecord`, `SupplyRecordDetail` 테이블을 사용한다.

해당 테이블이 데이터베이스에 존재하지 않는 경우, `main.py` 실행 시 자동으로 생성되도록 구현하였다.

---

## 7. 기능 평가용 샘플 쿼리

`src/sample_queries.sql` 파일에는 데이터베이스 기능을 평가하기 위한 샘플 SQL 쿼리가 포함되어 있다.

주요 쿼리는 다음과 같다.

```text
- 각 매장별 가장 많이 판매된 상위 20개 제품
- 시·도별 가장 많이 판매된 상위 20개 제품
- 판매 실적이 우수한 상위 5개 매장
- 특정 제품 B가 제품 A보다 더 많이 판매된 매장 수
- 특정 제품과 함께 가장 많이 구매된 제품 상위 3개
- 재고 부족 상품 조회
- 자동 재주문 발주 대상 조회
- 공급업체별 미처리 발주 내역 조회
- 공급 처리 기록 조회
- 회원별 구매 금액 및 포인트 조회
```

---

## 8. 주의사항

`convenience.db` 파일은 반드시 `main.py`와 같은 폴더에 있어야 한다.

SQLite 데이터베이스 파일이 아닌 파일을 `convenience.db`라는 이름으로 저장하면 `file is not a database` 오류가 발생할 수 있다.

DB 파일의 테이블 구조가 `main.py`에서 요구하는 구조와 다르면 필요한 테이블이 없다는 오류가 발생할 수 있다.
